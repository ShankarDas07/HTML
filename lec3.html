File Path system

  1. Step 1: The First Principle (The Fundamental Truth): The most fundamental truth is this: A website is not one single file.

    • It's a collection of different files (HTML, CSS, JavaScript, images, videos, fonts) that all need to work together. These files are organized into folders, just like documents on your computer.


  2. Step 2: The Core Problem: If your index.html file needs to display an image named logo.png, how does the HTML file tell the browser where to find logo.png?

    • You can't just say src="logo.png" and expect it to work every time. What if the logo is in an images folder? What if it's on a completely different website?

    • The browser needs an exact, unambiguous address to locate the file. A file path is that address.


  3. Relative File Paths AND Absolute File Paths

    A) Relative File Paths: A relative file path gives directions to a file starting from the location of the file you are currently in. You will use this 99% of the time for linking your own files together (images, CSS, other HTML pages).

    • Scenario 1: Linking to a file in the SAME folder.
      • The Problem: You are in index.html and want to link to about.html.
      • The Logic: They are neighbors, living in the same my-website/ folder.
      • The Syntax: You just use the filename.
      • Example:

        <a href="about.html">About Us</a>

    • Scenario 2: Linking to a file in a SUB-FOLDER (Going Down).
      • The Problem: You are in index.html and want to display logo.png.
      • The Logic: From index.html, you need to go into the images folder to find the logo.
      • The Syntax: Write the folder name, a forward slash /, and then the filename.
      • Example:

        <img src="images/logo.png" alt="My Website Logo">

    • Scenario 3: Linking to a file in a PARENT FOLDER (Going Up).
      • The Problem: You are in contact.html (inside pages/) and want to display logo.png (inside images/).
      • The Logic: Go up one level out of the pages folder to reach my-website/, then down into images/.
      • The Syntax: Two dots and a slash (../) means "go up one level."
      • Example:

        <img src="../images/logo.png" alt="My Website Logo">

    B) Absolute File Paths: An absolute file path gives the full, complete URL to a resource on the web. It starts with http:// or https://.

    When to use: Only when linking to a resource NOT on your own website.

    • Linking to another website
    • Using a font from Google Fonts
    • Using an image from an image-hosting service
    • Example:

      <a href="https://www.google.com">Search on Google</a>

      <img src="https://upload.wikimedia.org/wikipedia/commons/6/6a/Html5_logo_and_wordmark.svg" alt="HTML5 Logo">

  4. What is an HTML Boilerplate?

    An HTML boilerplate is the standard starting template for every HTML webpage. It provides all the basic instructions a browser needs before it displays your content. Without it, browsers may guess settings—but that can lead to unpredictable results.

    1. Why Do We Need It? A browser cannot assume:

      • Which HTML version you’re using
      • Which language the page is written in
      • How to scale the page for mobiles
      • How to decode characters like © or ₹

      If you just write <h1>Hello</h1>, the browser might still show it, but it will guess everything else, which can break layouts, encoding, or SEO.

      <html lang="en"> Sets the page language (for SEO & accessibility). Change “en” to “hi” for Hindi, etc.
      <head>...</head> Stores metadata (invisible setup info).
      <meta charset="UTF-8"> Uses UTF-8 character encoding so all symbols & languages display correctly.
      <meta name="viewport" content="width=device-width, initial-scale=1.0"> Makes the page responsive on mobiles.
      <title>Document</title> Sets the tab name and bookmark title.
      <body>...</body> Contains visible content (text, images, links, etc.).

  5. What is a Multipage Website?

    A multipage website splits content into separate pages (Home, About, Contact, etc.) instead of one long page. This keeps information organized and easy to navigate.

    1. Main Challenge: All pages must look connected and allow smooth movement between them.

    2. Key Solution:

      • Consistent Design: Use the same header, footer, and style on every page so the site feels unified.
      • Navigation Links: Add a menu using <a> (anchor) tags on every page to jump between pages.

  6. The <div> (The Generic Box)

    We need a way to group different elements (<h1>, <p>, <img>, etc.) into a single unit.

    1. Problem: Suppose you have a user profile card with an image, a heading, and a bio paragraph. Without grouping, you cannot apply a single border, background, or positioning to all of them at once.

    2. Solution: Use a generic, meaningless container called <div>. It lets you wrap related elements inside one box for styling or layout.

    Example Without div:

    < img src="avatar.png" >
    < h2>Arjun Kumar
    < p>Loves to code and teach HTML.< /p >

    Example With div:

    < div > < img src="avatar.png">
    < h2 >Arjun Kumar< /h2 >
    < p >Loves to codbre and teach HTML.< /p >< /div >


  7. class and id (The Labels for the Box)

    • First Principle : Once we have boxes (< div >s), we need a way to identify and find them.

    • The Core Problem : Your webpage might have ten different profile cards, a sidebar, a main content area, and a photo gallery—all made of 

      s. How do you tell the browser (specifically, your CSS or JavaScript) which box you want to style? How do you say, "Make all the profile cards have a gray background, but make the one featured profile card have a gold border?" You need a targeting system.

    • The Logical Solution: Two Types of Labels : The logical solution is to invent two types of labels (which we call attributes) that you can add to any HTML tag.

      1. A Unique Identifier: We need a label for one, and only one, specific element on the entire page. It must be unique. This is perfect for major, one-of-a-kind layout sections like the main navigation bar or a search form. This is the id.
      2. A Reusable Classifier: We also need a label that we can apply to multiple elements to group them into a category. This is for things that have a similar style or function, like all the profile cards, all the error messages, or all the "buy now" buttons. This is the class.

      Analogy

      • An id is like a person's unique Social Security Number or Aadhaar Number. Only one person can have it.
      • A class is like a person's Job Title (e.g., "Engineer"). Many people in a company can have the class "Engineer".

      This is the < div > (short for "division").

  8. The < span > (The Inline Highlighter)

    • First Principle :
      Sometimes we need to label and style a piece of content within a line of text, without creating a new block.

    • The Core Problem :
      What if you want to make just a single word inside a paragraph red?
      < p >This is very important information.< /p >
      You need to select the word "important". If you wrap it in a < div >, it will create a line break, ruining the sentence:
      < p >This is very < div >important< /div > information.< /p > --> Renders incorrectly.

    • The Logical Solution: The Generic Inline Container :
      We need a container that works just like a < div >, but is inline, meaning it does not create a line break. It just wraps a small piece of content within the natural flow of the text.
