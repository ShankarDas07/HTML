Basic of Internet

what is internet ?

Internet is a global communication system that links together thousands of individual networks. It allows exchange of information between two or more computers on a network. Thus internet helps in transfer of messages through mail, chat, video & audio conference, etc.



what is web ?

The web, or World Wide Web, is a global network of interconnected public webpages and digital resources accessible over the Internet. It functions as a vast information system that uses hyperlinks, Uniform Resource Locators (URLs), and protocols like HTTP to allow users to find and share information, such as text, images, audio, and video, through a web browser. Created by Tim Berners-Lee in 1989, the Web is not the same as the Internet but is one of its most prominent applications.


    1. Principle 1: The Need to Share Information :
      • The most fundamental truth is that we have information on one computer (Computer A) and we want it to be on another computer (Computer B). These computers are not in the same room.
      • How do we solve this? The simplest, most primitive way is to physically carry the information (e.g., on a USB stick). This works, but it's slow and doesn't scale. We need a direct, electronic connection.
      • So, we run a cable (like an ethernet cable or a fiber optic line) between Computer A and Computer B. We can now send electrical signals or pulses of light down this cable. We can agree on a simple code: a high-voltage signal means "1," and a low-voltage signal means "0."
      • We have just created the most basic form of a network: a direct link. We can now send bits (1s and 0s) between two computers.

    2. Principle 2: The Problem of Scale :
      • Scaling Problem : With 3 or more computers, direct cables between every pair grow rapidly (100 computers need 4,950 cables). This is impossible to manage.
      • Central Device : The solution is a Switch—a single device that every computer connects to.
      • How It Works : When Computer A wants to talk to Computer C, it sends the message to the switch, and the switch forwards it only to Computer C.
      • Local Network : This setup creates a Local Network (LAN) where nearby computers (home, office, school) can communicate efficiently.

    3. Principle 3: The Problem of Connecting Networks :

      When two offices each have their own Local Network, we can’t connect every computer in one office directly to every computer in the other.
      The solution is a Router— a special device that links one network to another and passes messages between them.
      By connecting routers together, many networks can communicate.
      This creates the Internet, which is simply a network of networks, all connected through routers.

      Router

    4. Principle 4: The Problem of Finding Anyone (Addressing) :

      To send a message across the world, every device needs a unique address, just like every house needs a unique street address.
      This is why we use an IP Address (Internet Protocol Address)—a unique number (e.g., 142.250.184.142) for every device on the Internet.
      When a message is sent, routers read the destination IP and forward it to the next best network step by step until it reaches the correct device.


    5. Principle 5: The Problem of Reliability and Fairness :

      Sending a huge file (like a 1-hour video) all at once is slow and risky—
      1) It would block the network for others.
      2) A single error would ruin the whole transfer.

      The solution is to split the file into many small, numbered pieces called Packets. Each packet carries:
      1) A chunk of data (payload)
      2) Destination IP (where it’s going)
      3) Sender IP (where it came from)
      4) A sequence number to reassemble in order.
      Packets travel independently across the Internet and may take different routes. The receiver collects, checks, and reassembles them to recreate the file. This system of sending and reassembling packets follows the TCP/IP protocol (Transmission Control Protocol / Internet Protocol)—the core rules of the Internet.


    6. IP Address : IPv4 (Internet Protocol version 4) is the traditional IP address format.
      • 1) Structure: A 32-bit number split into 4 sections (each 0–255) for human readability.
        Example: 172.217.16.142 (Google’s address)
      • 2) Limit: 32 bits allow only 2³² ≈ 4.3 billion unique addresses.
      • 3) Problem: When the Internet was created, 4.3 billion seemed enough, but with billions of devices (phones, laptops, servers, IoT), we are running out.

        • This shortage led to the use of public and private IPs to manage available addresses efficiently.

    7. Public IP Address : A Public IP address is your global, unique address on the internet.
      • Analogy : The company's main, public phone number.
      • Uniqueness: It must be globally unique. No two devices on the internet can have the same public IP address at the same time.
      • Purpose: To be reachable from anywhere in the world. This is the address that web servers, email servers, and any other public service use.
      • Assignment: It is assigned to you by your Internet Service Provider (ISP) (Comcast, AT&T, etc.). You don't control it; you lease it from them.
      • Who has one?  Your home router has one. The server hosting google.com has one. The server hosting Netflix has one.

    8. Private IP Address : A Private IP address is a local address used only within your own private network (e.g., your home Wi-Fi, your office network).
      • Example/Analogy: Like an employee’s internal extension number.
      • Uniqueness: Must be unique within the same network only
        • Your laptop can be 192.168.1.100 at home, and my laptop can have the same address in my home—no conflict because networks are separate.
      • Purpose: Lets devices on the same network communicate (e.g., laptop to printer, phone to smart TV) without needing a global IP.
      • Assignment: Your router assigns private IPs to devices.
      • Reserved Ranges:
        • 10.0.0.0 – 10.255.255.255 (large networks)
        • 172.16.0.0 – 172.31.255.255 (medium networks)
        • 192.168.0.0 – 192.168.255.255 (common for homes)

    9. IPv6 : IPv6 (Internet Protocol version 6) is the next generation of the Internet Protocol. Its primary purpose was to solve the address exhaustion problem of IPv4.

      The fundamental difference is the size of the address:

      • IPv4: Uses a 32-bit address, giving us ~4.3 billion unique addresses.
      • IPv6: Uses a 128-bit address.

        • The difference between 32-bit and 128-bit is not 4x. It's an exponential leap that is difficult to comprehend.The number of possible IPv6 addresses is 2^128, which is roughly:(340 undecillion)

    10. MAC Address: A MAC (Media Access Control) Address  is a unique, permanent serial number burned into every network-capable piece of hardware (your laptop's Wi-Fi card, your phone, your smart TV, the network port on your desktop).

      To track you, if you get offline and someone get the ip, they will get your message.

      A MAC address is a 48-bit number. To make it readable for humans, it's typically written as 12 hexadecimal digits.

      The most common ways you'll see it displayed are:

      1. Colon-Separated (Most Common): 3C:22:FB:A3:B4:C5
      2. Hyphen-Separated (Common on Windows): 3C-22-FB-A3-B4-C5
      3. Period-Separated (Used by Cisco and other network gear): 3c22.fba3.b4c5
      4. No Separators (Less common, seen in some software): 3C22FBA3B4C5

    11. Port Number : A Port number is a numeric identifier (0–65,535) used in networking to identify specific services or applications on a device.It works with an IP address to direct data to the correct service.

      Example: HTTP → Port 80, HTTPS → Port 443

      Technical Details:

      • 16-bit number: Stored using 16 binary digits.
      • Unsigned: Always positive.
      • Range: 0 – 65,535 (because 2¹⁶ = 65,536 total possibilities).
